  • Tools
    • REST API Documentation
      • Documentation has moved from just being static and has started providing a live API experience
      • Any product, if its capabilities are not clear for the consumers of the API, will find it hard to succeed
      • Providing just an API with bunch of example requests and responses is hardly useful
      • should provide enough business context around the functionality that their API is useful for and the problems that clients can solve using the API
      • REST is a style and not a standard
      • role-based access control (RBAC) - Details of which calls are allowed for which roles and how to obtain those roles is always helpful
      • Softwares
        • wadl
          • Complex to understand
          • Lack of toolings
        • Swagger from SmartBear: new Open API Initiative (OAI) or OpenApi
          • Swagger is a specification and complete framework implementation for describing, producing, consuming, and visualizing RESTful web services. Swagger is language agnostic
          • Swagger UI is a dependency-free collection of HTML, Javascript, and CSS assets that dynamically generate beautiful documentation from a Swagger-compliant API
          • Swagger CodeGen is a template-driven engine to generate client code in different languages using your Swagger documentation
          • The Swagger Ecosystem provides support for various tooling built both by external parties and the developers of the Swagger specification
        • RAML RESTful API Modeling Language from MULESOFT
          • RAML is a language for the definition of HTTP-based APIs that embody most or all of the principles of Representational State Transfer (REST)
          • RAML is also extremely popular and has a rich ecosystem of tooling built around RAML available - like to API Designer, API Workbench, RAML for JAX-RS etc
        • API Blue Print from APIARY
          • API Blueprint is a documentation-oriented web API description language
